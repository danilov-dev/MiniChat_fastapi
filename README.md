# MiniChat FastAPI - Практика работы с WebSocket

Простой чат на основе WebSocket, реализованный с использованием фреймворка FastAPI. Проект создан в учебных целях для демонстрации работы с WebSocket в Python.

## 📋 Описание

Это веб-приложение позволяет пользователям:
- Присоединяться к чат-комнатам по ID
- Отправлять и получать сообщения в реальном времени
- Видеть уведомления о присоединении/выходе участников
- Различать свои и чужие сообщения (визуальное оформление)

## 🛠 Технологии

- **Backend**: FastAPI (Python 3.12+)
- **WebSocket**: Встроенная поддержка FastAPI/Starlette
- **Frontend**: HTML, JavaScript (Vanilla), TailwindCSS
- **Шаблоны**: Jinja2
- **Сервер**: Uvicorn (ASGI)

## 🚀 Установка и запуск

### Требования
- Python 3.12 или выше
- Poetry (менеджер зависимостей)

### Шаг 1: Клонирование и установка зависимостей

```bash
cd /workspace
poetry install
```

### Шаг 2: Запуск сервера

```bash
poetry run uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

Параметры:
- `--reload`: Автоматическая перезагрузка при изменении кода (для разработки)
- `--host 0.0.0.0`: Доступ с любых сетевых интерфейсов
- `--port 8000`: Порт сервера

### Шаг 3: Открытие в браузере

Перейдите по адресу: `http://localhost:8000`

## 📖 Как использовать

1. **Вход в чат**:
   - Откройте `http://localhost:8000` в браузере
   - Введите ваше имя пользователя
   - Укажите ID комнаты (любое число >= 1)
   - Нажмите "Войти в чат"

2. **Общение**:
   - После подключения вы увидите уведомления о своём присоединении
   - Введите сообщение в поле ввода и нажмите Enter или кнопку "Отправить"
   - Сообщения отображаются в реальном времени
   - Ваши сообщения выделены синим цветом справа
   - Сообщения других пользователей — серые слева

3. **Мультиплеер**:
   - Откройте несколько вкладок браузера (или разных браузеров)
   - Войдите в одну и ту же комнату с одинаковым `room_id`
   - Сообщения будут синхронизироваться между всеми участниками

## 🔌 WebSocket API

### Подключение к комнате

```
ws://localhost:8000/ws/chat/{room_id}/{user_id}?username={username}
```

**Параметры:**
- `room_id` (int): ID комнаты чата (путь)
- `user_id` (int): Уникальный ID пользователя (путь)
- `username` (str): Имя пользователя (query-параметр)

### Формат сообщений

**От сервера (JSON):**
```json
{
  "text": "Текст сообщения",
  "is_self": true/false
}
```

**От клиента (текст):**
```
Просто текстовое сообщение
```

## ⚠️ Известные ограничения

1. **Генерация user_id**: Пользовательский ID генерируется случайно (1-10000). При перезагрузке страницы пользователь получит новый ID.

2. **Хранение подключений**: Активные подключения хранятся в памяти. После перезапуска сервера все подключения сбрасываются.

3. **Отсутствие аутентификации**: Любой пользователь может присоединиться к любой комнате.

4. **Нет истории сообщений**: При подключении новые пользователи не видят предыдущие сообщения.

5. **Однопоточность**: Менеджер соединений не использует блокировки, что может вызвать проблемы при высокой нагрузке.
